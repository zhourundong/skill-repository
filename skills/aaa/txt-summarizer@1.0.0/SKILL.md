# 使用指南

## 使用脚本预处理文本

在总结之前，先运行文本分析脚本获取统计信息：

```bash
python3 scripts/text_analyzer.py <txt文件路径>
```

脚本会返回 JSON 格式的分析结果，包含：
- 文本统计（字符数、词数、段落数等）
- 高频关键词（自动过滤停用词）
- 内容预览（前500字符）

## 总结流程

### 第一步：理解文本特征

根据脚本返回的统计信息，判断：
- **文本规模**：字符数 < 1000 为短文本，1000-5000 为中等，> 5000 为长文本
- **语言类型**：查看 chinese_characters 数量判断中英文比例
- **结构特点**：段落数和行数判断文本组织方式

### 第二步：阅读文本内容

使用 Read 工具读取完整文件内容。如果文件较大（>10000字符），可分段读取。

### 第三步：选择总结策略

根据用户需求和文本特征选择合适的总结模式：

#### 模式 A：简洁总结（默认）

适用于：
- 用户未指定详细要求
- 短文本或中等长度文本
- 快速了解内容

**输出格式：**
```
## 文档主题
[一句话概括核心主题]

## 关键要点（3-5个）
• 要点1
• 要点2
• 要点3

## 主要内容
[100-200字概述]
```

#### 模式 B：详细分析

适用于：
- 用户要求深入分析
- 长文本或复杂内容
- 学术、技术类文档

**输出格式：**
```
## 文档概览
- 文件名：[文件名]
- 文本规模：[字符数]字符，[词数]词
- 主要语言：中文/英文/混合

## 核心主题
[主题描述]

## 详细内容总结
[分章节或段落详细总结，可包含小标题]

### [第一部分标题]
[内容总结]

### [第二部分标题]
[内容总结]

## 关键信息
- 重要数据：[列出数字、日期、名称等]
- 核心观点：[主要论点]
- 关键词：[从脚本结果中提取的top关键词]

## 结论与意义
[文本的结论或实际意义]
```

#### 模式 C：结构化提取

适用于：
- 技术文档、报告、论文
- 需要结构化信息提取

**输出格式：**
```
## 文档元信息
| 属性 | 内容 |
|------|------|
| 文件路径 | [路径] |
| 字符数 | [数量] |
| 段落数 | [数量] |
| 主要关键词 | [前5个关键词] |

## 内容结构
1. [章节1标题]
2. [章节2标题]
3. [章节3标题]

## 核心内容
### 概念定义
[提取的定义和概念]

### 关键数据
- [数据项1]
- [数据项2]

### 方法/步骤
1. [步骤1]
2. [步骤2]

### 结论
[文档结论]

## 待解决问题
[如有疑问或需要进一步分析的内容]
```

#### 模式 D：要点列表

适用于：
- 会议记录、笔记、清单类文本
- 用户需要快速提取条目

**输出格式：**
```
## 文档要点

**主要议题：** [核心主题]

**要点清单：**
- [ ] 要点1
- [ ] 要点2
- [ ] 要点3
- [ ] 要点4
- [ ] 要点5

**行动项：**
- [ ] 需要执行的任务1
- [ ] 需要执行的任务2

**参考信息：**
- 相关数据：[数值]
- 相关人员/部门：[名称]
- 时间节点：[日期]
```

### 第四步：生成总结

根据选择的模式，结合以下原则生成总结：

1. **准确性**：忠实原文，不添加主观推断
2. **完整性**：涵盖主要信息，不遗漏重要细节
3. **清晰性**：语言简洁，结构分明
4. **实用性**：突出有价值的信息和关键点

## 特殊情况处理

### 空文件或极短文本

如果字符数 < 50：
```
该文件内容很少，完整内容如下：

[直接展示原文]
```

### 格式混乱的文本

如果文本缺乏清晰结构：
- 按段落逻辑重新组织
- 提取关键句子
- 用列表形式呈现要点

### 多语言混合文本

如果中英文混杂：
- 同时统计中英文字符
- 关键词提取同时考虑两种语言
- 保持原文术语的专业性

### 编码错误

如果出现乱码，尝试：
1. 提示用户文件可能存在编码问题
2. 尝试常见编码：utf-8, gbk, gb2312
3. 如无法解决，说明问题并建议转换编码

## 质量检查

生成总结后，确认：
- ✓ 总结长度适中（不超过原文30%）
- ✓ 关键信息完整覆盖
- ✓ 语言流畅自然
- ✓ 格式清晰易读
- ✓ 无明显错误或遗漏

## 注意事项

1. 始终先运行脚本获取基础统计数据
2. 根据文本特征选择合适的总结模式
3. 长文本（>10000字符）考虑分段总结
4. 尊重原文，不歪曲或过度解读
5. 如用户有特定需求（如提取特定信息），优先响应
